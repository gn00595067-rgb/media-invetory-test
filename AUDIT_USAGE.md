# 📊 檔次稽核功能使用說明

## 功能概述

檔次稽核功能會自動比對 Excel CUE 表與 Ragic 資料庫中的訂單資訊，驗證檔次（spots）是否正確。此功能使用經過驗證的 V29 演算法，達到 100% 準確率。

## 使用步驟

### 1. 準備 Excel 檔案

將需要稽核的 Excel CUE 表檔案放在指定資料夾中：
- 預設資料夾：`cue_downloads`（在專案根目錄）
- 支援格式：`.xlsx`、`.xls`
- 系統會自動搜尋資料夾內的所有 Excel 檔案（包含子資料夾）

### 2. 執行稽核

1. 開啟 Streamlit 應用程式
2. 切換到 **「📊 檔次稽核」** 頁籤
3. 在「Excel 檔案資料夾路徑」中輸入資料夾路徑（或使用預設值 `cue_downloads`）
4. 點擊 **「🚀 開始稽核」** 按鈕
5. 等待系統處理完成

### 3. 查看結果

稽核完成後，系統會顯示：

- **關鍵指標**：
  - 總檔案數
  - ✅ 成功數（檔次完全正確）
  - ⚠️ 差異數（檔次有差異但可接受）
  - ❌ 失敗數（無法驗證）
  - 🔴 高風險數（需要立即處理）

- **詳細結果表格**：
  - 以顏色標示風險等級
  - 顯示每個檔案的稽核結果

- **高風險案例**：
  - 自動篩選並顯示需要立即處理的案例

### 4. 下載報告

- **稽核報告 (CSV)**：包含所有稽核結果的詳細資料
- **詳細 Log (TXT)**：包含完整的處理過程記錄

## 結果說明

### 狀態欄位

- **成功**：Excel 表中的檔次與 Ragic 資料完全一致
- **差異**：檔次有差異，但差異在可接受範圍內（≤2）
- **失敗**：無法從 Excel 表中找到對應的檔次資訊

### 風險等級

- **低**：檔次完全正確，無需處理
- **中**：檔次有輕微差異（≤2），建議檢查
- **高**：檔次差異較大或無法驗證，需要立即處理

### 求解方法

系統使用多種演算法來求解檔次：

- **RowSolver**：從行總和求解
- **MultiUnit**：從多個單位值組合求解
- **UnitVal**：從單一單位值求解
- **BlockSolver**：從整個工作表掃描求解

## 注意事項

1. **檔案命名**：建議在 Excel 檔案名稱中包含訂單ID，以便系統自動比對
2. **資料夾路徑**：可以使用相對路徑（相對於專案根目錄）或絕對路徑
3. **Ragic 連線**：確保已正確設定 Ragic API Key（在 Streamlit Secrets 或環境變數中）
4. **處理時間**：大量檔案可能需要較長處理時間，請耐心等待

## 疑難排解

### 問題：找不到對應訂單

**原因**：Excel 檔案名稱中沒有包含訂單ID，或訂單ID格式不符

**解決方案**：
- 確認檔案名稱包含正確的訂單ID
- 或手動在 Ragic 中確認訂單是否存在

### 問題：求解失敗

**原因**：Excel 表中的資料格式不符合預期，或檔次資訊不明確

**解決方案**：
- 檢查 Excel 表格式是否正確
- 確認表中包含檔次相關的數值資訊
- 查看詳細 Log 了解具體原因

### 問題：無法載入 audit_core 模組

**原因**：`audit_core.py` 檔案不存在或路徑錯誤

**解決方案**：
- 確認 `audit_core.py` 檔案在專案根目錄
- 重新啟動 Streamlit 應用程式

## 技術細節

### 核心演算法

系統使用 V29 演算法，包含以下核心邏輯：

1. **Row Signature Extraction**：從 Excel 行中提取數值簽名
2. **Row Sum Solver**：從行總和求解目標值
3. **Multi-Unit Solver**：從多個單位值組合求解
4. **Unit Value Solver**：從單一單位值求解
5. **Block Fallback**：全表掃描備援方案

### 資料過濾

系統會自動過濾：
- 噪音行（包含金額、日期等無關資訊）
- 店數行（包含門市數量資訊）
- 黑名單數值（如 5、10、15 秒等常見誤判值）
