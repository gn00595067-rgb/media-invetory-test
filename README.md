# Ragic 秒數控管系統

## 功能特色

- 🔄 自動從 Ragic API 下載行銷排程資料
- 📊 將訂單裂解為每日庫存明細
- 📈 即時監控各平台秒數使用狀況
- 🎨 視覺化熱力圖顯示庫存狀態
- ⚙️ 可自訂平台店數與營業時間

## 安裝步驟

1. 安裝 Python 套件：
```bash
pip install -r requirements.txt
```

2. 執行系統：
```bash
streamlit run ragic_inventory.py
```

3. 瀏覽器會自動開啟，預設網址：http://localhost:8501

## 使用說明

### 首次使用
1. 點擊左側「🔄 強制從 Ragic 更新」按鈕下載資料
2. 在「📝 平台設定」中設定各平台的店數與營業時間
3. 系統會自動計算每日產能上限

### 主要功能頁面

- **表1-資料**：查看原始訂單資料
- **表2-秒數明細**：查看每日秒數使用明細（可篩選平台與日期）
- **表3-庫存熱力圖**：視覺化顯示各平台庫存狀態
  - 🟢 綠色：使用率 < 60%（安全）
  - 🟡 淺黃：使用率 60-80%（注意）
  - 🟠 黃色：使用率 80-100%（警戒）
  - 🔴 紅色：使用率 > 100%（超賣）
- **總結表**：查看月度與年度統計

## 設定說明

### 平台店數設定
在側邊欄的「平台設定」中，可以為每個平台設定：
- **店數**：該平台的總店數
- **每日營業小時數**：用於計算每日最大容量

系統會自動計算：`每日容量 = 店數 × 營業小時 × 3600秒`

## 注意事項

- 系統會自動檢查資料是否超過 24 小時，並提示更新
- 所有設定會儲存在本地 SQLite 資料庫中
- 建議每日至少更新一次資料以確保準確性

## 技術架構

- **前端框架**：Streamlit
- **資料處理**：Pandas
- **資料庫**：SQLite
- **API 串接**：Ragic REST API
